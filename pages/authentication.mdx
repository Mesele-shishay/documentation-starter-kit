# Authentication

## Overview

The M-Pesa SDK uses OAuth 2.0 for authentication. This guide explains how to handle authentication and manage access tokens.

## Basic Authentication

### Initial Setup

```php
use MesaSDK\PhpMpesa\Config;
use MesaSDK\PhpMpesa\Mpesa;

$config = new Config();
$config->setBaseUrl("https://apisandbox.safaricom.et")
    ->setConsumerKey("your_consumer_key")
    ->setConsumerSecret("your_consumer_secret")
    ->setEnvironment('sandbox');

$mpesa = new Mpesa($config);
```

### Authenticating

```php
try {
    $mpesa->authenticate();
    // Authentication successful
} catch (AuthenticationException $e) {
    // Handle authentication error
    echo "Authentication failed: " . $e->getMessage();
}
```

## Token Management

### Access Token

The SDK automatically manages access tokens. You can access the current token:

```php
$token = $mpesa->getAuth()->getAccessToken();
```

### Token Expiration

Tokens expire after a certain period. The SDK handles token refresh automatically:

```php
// Check if token is expired
if ($mpesa->getAuth()->isTokenExpired()) {
    // Token will be refreshed automatically on next request
    $mpesa->authenticate();
}
```

## Security Best Practices

### 1. Environment Variables

Always use environment variables for credentials:

```php
$config->setConsumerKey(getenv('MPESA_CONSUMER_KEY'))
    ->setConsumerSecret(getenv('MPESA_CONSUMER_SECRET'));
```

### 2. SSL Verification

Enable SSL verification in production:

```php
$config->setVerifySSL(true);
```

### 3. Error Handling

Implement proper error handling:

```php
try {
    $mpesa->authenticate();
} catch (AuthenticationException $e) {
    // Handle authentication errors
    error_log("Authentication Error: " . $e->getMessage());
} catch (NetworkException $e) {
    // Handle network errors
    error_log("Network Error: " . $e->getMessage());
} catch (Exception $e) {
    // Handle other errors
    error_log("Unexpected Error: " . $e->getMessage());
}
```

### 4. Token Storage

For production applications, consider implementing custom token storage:

```php
class CustomTokenStorage implements TokenStorageInterface {
    public function saveToken($token, $expiresIn) {
        // Store token securely
    }

    public function getToken() {
        // Retrieve stored token
    }

    public function hasToken() {
        // Check if token exists
    }
}

$config->setTokenStorage(new CustomTokenStorage());
```

## Common Issues

### 1. Invalid Credentials

- Verify your consumer key and secret
- Check if credentials are properly set
- Ensure environment matches credentials

### 2. Network Issues

- Check internet connectivity
- Verify API endpoint accessibility
- Check firewall settings

### 3. SSL Issues

- Ensure valid SSL certificates
- Check certificate chain
- Verify SSL configuration

## Production Considerations

### 1. Rate Limiting

Handle rate limiting gracefully:

```php
try {
    $mpesa->authenticate();
} catch (RateLimitException $e) {
    // Implement exponential backoff
    sleep(pow(2, $e->getRetryCount()));
    $mpesa->authenticate();
}
```

### 2. Monitoring

Implement monitoring for authentication issues:

```php
$logger = new MpesaLogger();
$logger->setLogPath('/path/to/logs');
$mpesa->setLogger($logger);
```

### 3. Security Headers

Add security headers to requests:

```php
$config->setHeaders([
    'X-Request-ID' => uniqid(),
    'X-Client-Version' => '1.0.0'
]);
```

## Testing

### 1. Sandbox Environment

Use sandbox environment for testing:

```php
$config->setEnvironment('sandbox')
    ->setBaseUrl("https://apisandbox.safaricom.et");
```

### 2. Mock Authentication

For testing, you can mock authentication:

```php
$mockAuth = new MockAuthentication();
$config->setAuthentication($mockAuth);
```
