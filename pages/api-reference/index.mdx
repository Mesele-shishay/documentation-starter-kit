# API Reference

This section provides detailed documentation for all public APIs, classes, and methods available in the project.

## Core Classes

### Class Overview

Here's a quick overview of the main classes in the project:

| Class             | Description                       | Namespace              |
| ----------------- | --------------------------------- | ---------------------- |
| `MainClass`       | Core functionality implementation | `App\Core`             |
| `ServiceProvider` | Service provider implementation   | `App\Providers`        |
| `Controller`      | Base controller class             | `App\Http\Controllers` |

## Namespaces

The project is organized into the following main namespaces:

- `App\Core` - Core functionality
- `App\Http` - HTTP-related components
- `App\Providers` - Service providers
- `App\Models` - Data models
- `App\Services` - Service layer implementations

## API Endpoints

### Authentication

```php
POST /api/auth/login
POST /api/auth/register
POST /api/auth/logout
```

### Data Management

```php
GET /api/data
POST /api/data
PUT /api/data/{id}
DELETE /api/data/{id}
```

## Error Handling

All API endpoints follow a consistent error handling pattern:

```json
{
  "status": "error",
  "code": 400,
  "message": "Error description",
  "details": {}
}
```

## Rate Limiting

API requests are subject to rate limiting:

- Authenticated requests: 60 requests per minute
- Unauthenticated requests: 30 requests per minute

## Authentication

All authenticated endpoints require a valid JWT token in the Authorization header:

```http
Authorization: Bearer <your-jwt-token>
```
